────────────────────────────────────────────────────────────────────────────────
١. تدفق الطلب (Request Flow)
────────────────────────────────────────────────────────────────────────────────

  المستخدم/التطبيق
        |
        |  GET/POST/PUT/PATCH/DELETE  +  Bearer Token (إن وُجد)
        v
  ┌─────────────────┐
  |  Middleware     |  throttle → auth:sanctum → role → permission (حسب المسار)
  └────────┬────────┘
           v
  ┌─────────────────┐
  |  Controller     |  يستدعي Request (تحقق) و Service (منطق)
  └────────┬────────┘
           v
  ┌─────────────────┐     ┌─────────────────┐
  |  Service        | --> |  Model(s)       |  استعلامات وعلاقات
  └────────┬────────┘     └────────┬────────┘
           |                        v
           |                 ┌─────────────────┐
           |                 |  Database       |
           |                 └─────────────────┘
           v
  ┌─────────────────┐
  |  Resource       |  تحويل الموديل إلى مصفوفة/JSON
  └────────┬────────┘
           v
  استجابة JSON للعميل


────────────────────────────────────────────────────────────────────────────────
٢. توزيع المسارات حسب الأدوار
────────────────────────────────────────────────────────────────────────────────

  بدون مصادقة:
    POST /login
    GET  /csrf-token

  مصادق (auth:sanctum) فقط:
    /user, /logout
    /project_management/teams/index
    /developers, /developers/{id}
    /ai/* (مع throttle:ai-assistant)
    /contracts/* (عرض/إنشاء/تحديث/حذف حسب السياسة)
    /notifications/*
    /my-tasks/*, /tasks/sections/*
    /chat/*
    /teams/index, /teams/show/{id}
    /exclusive-projects/*
    /storage/{path}

  حسب الدور:
    project_management|admin  --> عقود إدارية، طرف ثاني، وحدات، أقسام، لوحة، فرق
    editor|admin             --> عقود، وحدات، أقسام (مونتاج، تصوير، لوحات)
    sales|sales_leader|admin  --> لوحة مبيعات، مشاريع، حجوزات، أهداف، حضور، قائمة انتظار، تفاوض، خطط دفع
    marketing|admin          --> لوحة تسويق، مشاريع، خطط، مهام، تقارير، إعدادات، leads
    accounting|admin         --> لوحة محاسبة، عمولات، ودائع، رواتب، ملفات مطالبات
    credit|admin             --> لوحة ائتمان، حجوزات، تمويل، نقل ملكية، ملفات مطالبات
    hr (middleware)          --> مستخدمون، فرق، تقارير، لوحة، أداء مسوقين
    inventory (middleware)    --> عقود، طرف ثاني، وحدات، لوحة، مواقع
    admin                    --> موظفون، عقود إدارية، إشعارات، تعيين مشاريع مبيعات

  AI:
    ai/ask, ai/chat, ai/conversations, ai/sections  (مصادق + throttle)
    ai/knowledge/*   (admin)
    ai/assistant/chat (مصادق)
    ai/calls/*       (admin|sales|sales_leader|marketing + permission)
  Webhooks (بدون auth مستخدم):
    webhooks/twilio/*  (تحقق توقيع Twilio)


────────────────────────────────────────────────────────────────────────────────
٣. علاقة الوحدات بالبيانات (مبسطة)
────────────────────────────────────────────────────────────────────────────────

  Contract ──┬── ContractInfo (طرف ثاني)
             ├── ContractUnit (وحدات)
             ├── BoardsDepartment / PhotographyDepartment / MontageDepartment
             ├── SalesReservation (حجوزات)
             ├── SalesProjectAssignment (تعيين مشاريع مبيعات)
             └── Teams (ربط فرق)

  SalesReservation ──┬── ReservationPaymentInstallment (أقساط)
                     ├── NegotiationApproval
                     ├── SalesReservationAction
                     ├── Deposit
                     ├── Commission
                     └── CreditFinancingTracker / TitleTransfer / ClaimFile

  User ──┬── Roles & Permissions (Spatie)
         ├── EmployeeContract, EmployeeWarning
         ├── SalesTarget, SalesTeamMemberRating
         ├── Marketing (DeveloperPlan, EmployeePlan, MarketingTask, …)
         ├── AccountingSalaryDistribution, CommissionDistribution
         └── AIConversation, Message (دردشة)
